핵심 개념 마스터

경우의 수 문제를 풀려면 이 7개 개념이 기본이야. 하나씩 확실히 잡아두자.

1

합의 법칙 vs 곱의 법칙 (OR vs AND)

경우의 수의 가장 기본이 되는 두 가지 법칙이야. 이 둘만 정확히 구분할 수 있으면 나머지는 전부 여기서 파생된 거야.

합의 법칙 (OR)

"A 또는 B 중 하나를 선택"

m + n

동시에 일어나지 않는 두 사건

키워드: "또는", "~이거나", "~중 하나"

곱의 법칙 (AND)

"A를 하고 그리고 B도 하기"

m × n

연달아(순서대로) 일어나는 두 사건

키워드: "그리고", "연이어", "동시에", "~한 후"

시각적 이해

합의 법칙 — 갈림길에서 하나 고르기
출발
|
어디로 갈까? (OR)
길 A (3가지) 길 B (4가지)
총 = 3 + 4 = 7가지
곱의 법칙 — 순서대로 선택하기
1단계
3가지
→
2단계
4가지

총 = 3 × 4 = 12가지
예제

서울에서 부산까지 가는 방법: 비행기 2편, KTX 5편, 버스 3편이 있다.

하나를 골라 가는 거니까 → 합의 법칙
2 + 5 + 3 = 10가지

서울 → 대전 → 부산: 서울~대전 3가지, 대전~부산 4가지.

순서대로 이동하는 거니까 → 곱의 법칙
3 × 4 = 12가지

!
합 vs 곱 판별 꿀팁 "그 중 하나만 하면 끝" → 합의 법칙 (OR)
"이것도 하고 저것도 해야 끝" → 곱의 법칙 (AND)
헷갈리면 문장을 "그리고"/"또는"으로 바꿔 읽어봐.
심화 예제

주사위 2개를 던져서 합이 6인 경우의 수

(1,5), (2,4), (3,3), (4,2), (5,1) → 각 조합은 OR 관계.
하지만 각 조합 안에서 "첫째 주사위 = 1 AND 둘째 주사위 = 5"는 곱의 법칙.
여기선 각 조합이 정해진 거니까 결국 합의 법칙으로 5가지.

순열 (Permutation) — 순서가 있는 나열

n개 중에서 r개를 뽑아 순서대로 나열하는 경우의 수야. "누가 1등이고 누가 2등인지"가 중요하면 순열이야.

순열 공식 nPr = n! / (n-r)!
n개에서 r개를 순서 있게 뽑기

공식은 어디서 나왔을까?

5명 중 3명을 뽑아서 줄 세우는 걸 생각해보자.

5명 중 3명 줄 세우기 — 자리별 선택
5
1번째
×
4
2번째
×
3
3번째
=
60
총

1번째 자리: 5명 전원 가능 → 5
2번째 자리: 1명 빠졌으니 → 4
3번째 자리: 2명 빠졌으니 → 3

곱의 법칙으로 5 × 4 × 3 = 60.

이걸 팩토리얼로 쓰면: 5! / (5-3)! = 120 / 2 = 60. 같은 거야.

i
팩토리얼(!) 복습
n! = n × (n-1) × (n-2) × ... × 2 × 1
5! = 5×4×3×2×1 = 120
0! = 1 (약속)

특수한 경우

전체 순열 (n개 전부 나열) nPn = n!
n개 전부를 줄 세우는 경우 = n!
예제

7명 중 회장, 부회장, 총무를 뽑는 경우의 수

회장/부회장/총무는 서로 다른 직책이니까 순서가 있다 → 순열

7P3 = 7 × 6 × 5 = 210가지
예제

A, B, C, D, E 5명이 한 줄로 서는 경우의 수

5P5 = 5! = 5 × 4 × 3 × 2 × 1 = 120가지

조합 (Combination) — 순서 없이 뽑기

n개 중에서 r개를 순서 상관없이 뽑는 경우의 수야. "누구를 뽑았느냐"만 중요하고, 뽑는 순서는 상관없어.

조합 공식 nCr = n! / (r! × (n-r)!)
순열에서 중복 순서를 나눠 제거

순열 vs 조합 — 핵심 차이

A, B, C 3명 중 2명 뽑기
순열 (순서 O)
(A,B) ≠ (B,A)
(A,C) ≠ (C,A)
(B,C) ≠ (C,B)
= 6가지
조합 (순서 X)
{A,B}
{A,C}
{B,C}
= 3가지
조합 = 순열 ÷ (뽑은 r개의 순서 수) = 6 ÷ 2! = 3

공식 유도

순열은 "뽑고 + 줄 세우기"야. 조합은 "뽑기만" 하는 거야.
그래서:

nPr = nCr × r! // 순열 = 조합 × (뽑은 것을 줄 세우는 수) ∴ nCr = nPr / r!     = n! / ((n-r)! × r!)
!
순열 vs 조합 판별법
"뽑은 다음에 역할/순서/위치가 다르면" → 순열
"그냥 뽑기만 하면" → 조합

예: "대표 3명 선출" = 조합 / "회장, 부회장, 총무 선출" = 순열

유용한 성질

대칭성 nCr = nCn-r
파스칼 항등식 nCr = n-1Cr-1 + n-1Cr
예제

10명 중 4명으로 팀을 구성하는 경우의 수

팀은 역할 구분 없이 뽑기만 하면 되니까 → 조합

10C4 = 10! / (4! × 6!) = (10 × 9 × 8 × 7) / (4 × 3 × 2 × 1) = 5040 / 24 = 210가지
*
계산 꿀팁
nCr 계산할 때 분자분모를 바로 약분하면 빨라.
10C4 = (10×9×8×7) / (4×3×2×1) → 10/2=5, 9/3=3, 8/4=2 → 5×3×2×7 = 210

같은 것이 있는 순열

나열하려는 것들 중에 구분할 수 없는 것이 있을 때 쓰는 거야. 예를 들어 "BANANA"의 글자를 재배열하면, A끼리 서로 바꿔도 같은 단어야.

같은 것이 있는 순열 n! / (p! × q! × r! × ...)
n개 전체 중 같은 것이 p개, q개, r개, ...

왜 이렇게 될까?

전체 n개를 줄 세우면 n!가지인데, 구분 못 하는 것끼리의 자리 바꿈(p!가지)은 같은 결과를 만드니까 나눠주는 거야.

"AAB" 배열하기 — 같은 것 구분 실험

A를 A1, A2로 구분하면:

A1 A2 B → AAB
A2 A1 B → AAB
A1 B A2 → ABA
A2 B A1 → ABA
B A1 A2 → BAA
B A2 A1 → BAA

3! = 6가지인데 실제론 3가지뿐. → 3! / 2! = 6 / 2 = 3

예제

"BANANA"의 알파벳을 재배열하는 경우의 수

전체 6글자: B(1개), A(3개), N(2개)

6! / (1! × 3! × 2!) = 720 / (1 × 6 × 2) = 720 / 12 = 60가지
예제

같은 빨간 공 3개, 같은 파란 공 2개, 같은 노란 공 1개를 일렬로 나열

6! / (3! × 2! × 1!) = 720 / (6 × 2 × 1) = 60가지

원순열 (Circular Permutation)

원형으로 둘러앉는 경우의 수야. 직선 배열과 달리, 회전해서 같아지는 건 하나로 치니까 경우의 수가 줄어들어.

원순열 공식 (n-1)!
n명이 원형으로 앉기 (회전 동치 제거)

왜 (n-1)! 인가?

4명의 원탁 배치 — 회전 동치
A
B
C
D
D
A
B
C

이 두 배치는 시계방향으로 한 칸 회전한 것뿐 → 같은 배열로 취급

직선이면 4! = 24가지인데, 원형에서는 4개씩(회전) 같으니까 나눠줘야 해:
4! / 4 = 24 / 4 = 6 = (4-1)! = 3!

i
직관적 이해
원형이니까 한 명(예: A)을 고정시켜 놓고 나머지를 배열하면 돼. A를 고정 → 나머지 (n-1)명 배열 → (n-1)!
예제

6명이 원탁에 둘러앉는 경우의 수

(6-1)! = 5! = 5 × 4 × 3 × 2 × 1 = 120가지
!
원순열 주의사항
뒤집기(거울상)까지 같은 경우 (팔찌, 목걸이 등): (n-1)! / 2
원탁처럼 앞뒤가 있으면 (n-1)!, 팔찌처럼 뒤집을 수 있으면 (n-1)!/2

여사건 활용법 — "~하지 않는" 경우

"적어도 하나는 ~한 경우" 또는 "~하지 않는 경우"가 나오면 직접 세는 것보다 여사건이 훨씬 쉬운 경우가 많아.

여사건 공식 (구하는 것) = (전체) − (여사건)
"~하지 않는 경우" = 전체에서 "~하는 경우"를 빼기
여사건의 구조
전체 경우의 수 (U)
여사건
"~하는 경우"
구하는 것
"~하지 않는 경우"

여사건이 유리한 상황

*
다음 키워드가 보이면 여사건을 떠올려:
"적어도 하나" "~하지 않는" "모두 ~은 아닌" "최소한"
예제

주사위 2개를 던질 때, 적어도 하나가 6인 경우의 수

직접 세기: (하나만 6) + (둘 다 6) → 케이스가 여러 개라 복잡해.

여사건으로 풀면:

// 전체: 6 × 6 = 36 // 여사건: "둘 다 6이 아닌 경우" = 5 × 5 = 25 ∴ 36 - 25 = 11가지
예제

5명을 한 줄로 세울 때, A와 B가 이웃하지 않는 경우의 수

// 전체: 5! = 120 // 여사건: "A,B가 이웃하는 경우" // A,B를 묶어서 한 덩어리 → 4! × 2! = 48 ∴ 120 - 48 = 72가지
!!
흔한 실수
여사건을 쓸 때, "전체"가 정확한지 반드시 확인해. 전체 범위를 잘못 잡으면 답이 완전히 틀려져.

조건부 배정 — 제약조건이 있는 문제

"특정 사람은 특정 자리에 올 수 없다", "A는 반드시 B보다 먼저" 같은 제약조건이 붙은 경우의 핵심 전략이야.

3가지 핵심 전략

전략 1: 제약조건 있는 것 먼저 배치

조건이 걸린 원소를 가장 먼저 배치하고, 나머지를 그 다음에 배치해. 선택지가 적은 것부터 처리하는 게 핵심이야.

*
예: "A는 맨 앞 또는 맨 뒤에만 설 수 있다" → A를 먼저 배치(2가지) → 나머지 배치
전략 2: 묶어서 한 덩어리로

"~가 이웃해야 한다" → 이웃해야 하는 것들을 하나로 묶어서 전체를 배열한 후, 묶음 내부의 순서도 곱해줘.

*
예: "A,B,C가 연속" → [ABC]를 하나로 → 전체 배열 × 3!
전략 3: 경우 나누기 (Case Analysis)

조건이 복잡할 때, 가능한 경우를 케이스별로 나눠서 각각 계산하고 합산해. 케이스가 겹치지 않게 나누는 게 중요해.

*
예: "A가 업무1을 맡는 경우" vs "A가 업무1을 안 맡는 경우"
예제 — 전략 1

7명이 한 줄로 설 때, A는 양 끝에만 설 수 있다.

  1. A가 갈 수 있는 자리: 맨 앞 또는 맨 뒤 = 2가지
  2. 나머지 6명 배열: 6! = 720
  3. 곱의 법칙: 2 × 720 = 1440가지
예제 — 전략 2

5명이 한 줄로 설 때, A와 B가 반드시 이웃해야 하는 경우

  1. {A,B}를 하나의 블록으로 묶기 → 4개 배열 = 4! = 24
  2. 블록 내부에서 A,B 순서: 2! = 2
  3. 곱의 법칙: 24 × 2 = 48가지

1031 문제 풀이

실전 문제를 단계별로 풀어보자. 어떤 개념이 쓰이는지 명확하게 짚어줄게.

Q PROBLEM 1
남학생 3명과 여학생 3명이 한 줄로 설 때, 여학생 3명이 모두 이웃하지 않는 경우의 수를 구하여라.

풀이 전략 분석

i
핵심 판단: "이웃하지 않는" = 직접 세기 어려움 → 여사건 또는 사이 배치법 사용
여기선 "3명 전부 이웃하지 않는" 조건이라 사이 배치법이 깔끔해. 여사건으로도 풀 수 있지만 케이스가 복잡해져.

풀이: 사이 배치법 (Slot Method)

STEP 1

남학생 3명을 먼저 배치한다

M2
M3

남학생 3명을 한 줄로 세우는 경우: 3! = 6가지

남학생 사이사이 + 양 끝에 빈 자리를 확인한다

남학생이 서면, 여학생이 들어갈 수 있는 자리(슬롯)가 만들어져:

_

빈 자리(_)가 4개 생겨. 여학생끼리 이웃하지 않으려면 이 빈 자리에 한 명씩 넣으면 돼.

STEP 3

4개 슬롯 중 3개를 골라 여학생 배치

4개 빈 자리 중 3개를 선택하고 (순서 있게), 여학생을 배치해.

i
여기서 조합이 아니라 순열을 쓰는 이유: 여학생 3명은 서로 다른 사람이니까 "어느 자리에 누가 서는지"가 중요해. 즉 4P3.
STEP 4

곱의 법칙으로 최종 답 계산

// 남학생 배열 × 여학생 슬롯 배치 3! × 4P3 = 6 × 24 = 144
ANSWER
144가지

여사건으로도 풀어볼까?

여사건 풀이 펼치기

"여학생 3명이 모두 이웃하지 않는" 의 여사건은 "여학생 중 적어도 2명이 이웃하는"이야.

그런데 이게 케이스가 복잡해:

  • 정확히 2명만 이웃 (제3의 여학생은 떨어져있음)
  • 3명 모두 이웃

겹치는 부분도 고려해야 해서 포함-배제가 필요해져.

// 전체: 6! = 720 // Case A: 3명 모두 연속 // [F1,F2,F3]을 한 블록으로 → 4개 배열 × 블록 내부 // = 4! × 3! = 24 × 6 = 144 // Case B: 정확히 2명 연속 (3번째는 분리) // 여학생 2명 묶는 조합: C(3,2) = 3 // [F_i,F_j]를 블록으로 → 5개 중 나열 // 5! × 2! = 240 (각 페어당) // 근데 3번째 여학생이 블록 옆에 붙는 경우 빼야 해 ... // → 이렇게 복잡해지니까 사이 배치법이 훨씬 나아!

교훈: "아무도 이웃하지 않는" 문제에서는 사이 배치법이 왕이야. 여사건은 "적어도 하나가 ~인 경우"에 더 유리해.

이 문제에서 배울 점

*
"이웃하지 않는" 문제의 정석 = 사이 배치법
1) 제약 없는 쪽을 먼저 배치
2) 생긴 슬롯(빈칸)에 제약 있는 쪽을 배치
3) 곱의 법칙으로 곱하기

Q PROBLEM 2
6명의 회사원 A, B, C, D, E, F가 6가지 업무 (1)(2)(3)(4)(5)(6)을 한 사람당 하나씩 처리한다.

제약조건:
• 업무 (1)은 A와 B만 할 수 있다.
• 업무 (2)는 A, B, C만 할 수 있다.
• C는 업무 (5)를 처리할 수 없다.

업무를 나누는 방법의 수를 구하여라.

제약조건 정리

제약조건 매트릭스
사람 \ 업무 (1) (2) (3) (4) (5) (6)
A O O O O O O
B O O O O O O
C X O O O X O
D X X O O O O
E X X O O O O
F X X O O O O
i
전략: 선택지가 가장 적은 업무부터 배정해. 업무(1)은 2명만 가능, 업무(2)는 3명만 가능 → 이 순서대로 먼저 처리. 단, 업무(1)에 누가 가느냐에 따라 업무(2)의 선택지가 달라지니까 케이스 분리가 필요해.

STEP 1: 업무 (1) 배정 — 2가지 큰 케이스

업무(1)은 A 또는 B만 가능. 이걸 기준으로 나눠.

Case 1: A가 업무(1)을 맡는다
Step 1-1: 업무(2) 배정

업무(2)는 A, B, C만 가능한데, A는 이미 업무(1)에 배정됐어.
남은 후보: B, C → 2가지

Case 1-A: A→(1), B→(2)

남은 사람: C, D, E, F → 남은 업무: (3)(4)(5)(6)
C는 (5)를 할 수 없어. 나머지 제약 없음.

C의 선택지: (3)(4)(6) 중 하나 = 3가지
C 배정 후 남은 3명(D,E,F)이 남은 3개 업무를 자유롭게 배정: 3! = 6

Case 1-A = 3 × 6 = 18
Case 1-B: A→(1), C→(2)

남은 사람: B, D, E, F → 남은 업무: (3)(4)(5)(6)
제약 조건 없음! (C는 이미 배정됐고, B에 대한 추가 제약 없음)

Case 1-B = 4! = 24
Case 1 합계 = 18 + 24 = 42
Case 2: B가 업무(1)을 맡는다
Step 2-1: 업무(2) 배정

업무(2)는 A, B, C만 가능한데, B는 이미 업무(1)에 배정됐어.
남은 후보: A, C → 2가지

Case 2-A: B→(1), A→(2)

남은 사람: C, D, E, F → 남은 업무: (3)(4)(5)(6)
C는 (5) 불가.

C의 선택지: (3)(4)(6) 중 하나 = 3가지
나머지 3명이 3개 업무: 3! = 6

Case 2-A = 3 × 6 = 18
Case 2-B: B→(1), C→(2)

남은 사람: A, D, E, F → 남은 업무: (3)(4)(5)(6)
제약 조건 없음!

Case 2-B = 4! = 24
Case 2 합계 = 18 + 24 = 42

STEP 2: 최종 합산

// Case 1 (A→(1)) + Case 2 (B→(1)) = 42 + 42 = 84
ANSWER
84가지

검증: 왜 대칭인가?

Case 1과 Case 2가 같은 이유

A와 B는 업무(1)과 업무(2)에 대해 완전히 같은 자격을 가져. (둘 다 (1) 가능, 둘 다 (2) 가능, 나머지 업무에 대한 제약 없음)

그래서 "A→(1)"과 "B→(1)"의 구조가 대칭이 돼. 실전에서는 이걸 알아채면 한쪽만 계산하고 ×2 하면 돼서 시간을 절약할 수 있어.
